Page Structure / Element naming conventions

  • each page has the following structure:

    #header
#content
#footer

  • #content is marked with one of these page types:

    • .collection - for collections page, search page, blog page
    • .page - for product page, article page, not_found, and page
    • .index - for the start page
    • in any case the page template name is added as a class to #content

  • inside #content everything is wrapped into #section or #article

    #content.index
  #section.about
  #section.latest
  #section.featured
  ...
  #section.navigation

#content.product
  #article
  #section.navigation

  • naming elements is done by following the HTML5 specification:

    • #nav is used to identify all navigation elements (menus)
    • #article identifies every product or blog post

    • #article.thumb displays more then one articles in a collection, and has the following structure:

      #article.thumb
  #details
  #figure
  #figcaption
    #title
    #price
    #summary

    • in general #details, #figure, #figcaption and #summary is used whenever possible

Other conventions

  • accordion like navigation is marked up like:

    .navigation
  <h3>Title
  <ul> ...

  <h3>Title
  <ul> ...

  ...

  • every navigational element is wrapped into an <ul>
