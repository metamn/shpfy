Introduction

Gadgetoman is built on a fluid layout using the Responsive Web Design technique. This allows to be streched to various tablet and mobile displays.

All measuring units -- for containers, divs, text, images, etc -- are either in "%" or "em". No "px" is used for positioning. They are used only for decoration: borders, text shadows, etc.

There are different CSS files each dedicated to a separate task:

  • base.css -- To reset the browser and for holding some common styling like alignments, clearfixes etc.
  • layout.css -- The grid and the design elements aligned on it. Has no color definitions at all.
  • theme.css -- The current theme / color scheme, as seen at the webpage. Has no alignment definitions at all just colors.
  • mobile.css -- The stylesheet for mobile devices
  • tablet.css -- The stylesheet for tablets

Typography

  • Nothing special, inherited from base.css
  • Uses the Ubuntu Google font which is hardwired (the .js in header)
  • Plain text headings margin-bottom is moderated
    • h1, h2, h3 under .text has margin 0

Color scheme

  • basic color / background: #eee;
    • grays, from light to dark: #f9f9f9, #f3f3f3, #dadada, #ddd, #ccc, #a3a3a0, #777, #4c4c4c
  • black: #3d3d3b > 2d2d2d
  • highlight / orange: #ff3300
  • highlight / yellow: #C60017

Links

  • The default style is simple, no underline, no hover
    • a.underline forces text decoration (!important)
  • Plain text style for post body, product description, pages etc.
    • class name: .text
    • default style: underlined
    • hover: inverted colors
    • headings are unstyled using a.title
  • Notes, highlighting texts
    • class name: .note
    • default style: gray with shadow, changing color on hover, no decoration
